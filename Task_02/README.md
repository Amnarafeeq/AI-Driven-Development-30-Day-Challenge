# AI-Driven Development 

## Part A — Theory

### Q1: Nine Pillars Understanding

**1. AI development agents repetitive tasks ke liye behtar kyu hain?**  
AI agents boring aur repeat hone wale setup steps ko bohot fast handle kar lete hain. Is se developer ka time bach jata hai aur wo zyada architecture, planning aur system design par focus kar sakta hai. Is tarah developer dheere dheere sirf code writer nahi, balkay ek sochne wala system architect ban jata hai.

**2. Nine Pillars developer ko M-Shaped Developer kaise banate hain?**  
Nine Pillars developer ko tools, specs, agents aur testing ke saath ek complete flow sikha dete hain. Is process se developer aik se zyada domains (jaise coding, design, automation, agent orchestration) me depth banata hai. Yehi cheez usay M-Shaped developer banati hai — yani multiple strong skills combined.

---

### Q2: Vibe Coding vs Specification-Driven Development

**1. Vibe Coding ek hafte baad problem kyu ban jata hai?**  
Vibe coding me start strong hota hai lekin kuch din baad code messy ho jata hai kyun ke koi clear plan ya structure nahi hota. Cheezen modify karna mushkil hota hai, bugs barhte hain aur developer ko yaad nahi rehta ke pehle kya logic banaya tha.

**2. SDD in problems ko kaise rokta hai?**  
Specification-Driven Development me pehle hi clear rules aur structure likh liya jata hai. Is se code predictable hota hai, future me changes aasani se hoti hain aur koi bhi naye developer ko jaldi samajh aa jata hai. Whole system stable rehta hai.

---

### Q3: Architecture Thinking

**1. Architecture-first thinking developer ka role kaise change karta hai?**  
Architecture-first approach developer ko sirf coder nahi balkay decision-maker bana deti hai. Wo pehle system ka flow, layers aur agents ka behaviour decide karta hai, phir coding start hoti hai. Is se developer ki value bohot barh jati hai.

**2. Developers ko layers/systems me sochna kyu zaroori hai?**  
AIDD me sirf code likhne se kaam nahi hota. Developer ko samajhna hota hai ke model layer, workspace layer aur agent layer aik doosre se kaise interact karte hain. Layered soch se system scalable, clean aur maintainable banta hai.
